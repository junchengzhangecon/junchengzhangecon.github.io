# 怎么修改这个个人主页

这个网站是用 Quarto 做的。你平时不要直接改 `index.html`、`research.html`、`teaching.html` 这些 HTML 文件。

正确逻辑是：

1. 修改外面的 `.qmd` 源文件
2. 运行 `quarto render`
3. Quarto 自动生成新的 `.html` 网页文件
4. 用 git 提交并推送到 GitHub
5. GitHub Pages 自动更新网站

## 最常改的文件

- `index.qmd`：首页内容，也就是打开网站第一眼看到的页面
- `research.qmd`：Research 页面，写研究兴趣、研究方向
- `teaching.qmd`：Teaching 页面，写课程和教学经历
- `cv.qmd`：CV 页面，写教育经历、教学经历、奖项、语言等
- `contact.qmd`：Contact 页面，写邮箱、地址、链接
- `_quarto.yml`：网站总设置，包括顶部导航栏、网站标题、页脚
- `styles.css`：网站样式，比如颜色、字体、间距

## 例子 1：修改首页文字

打开：

`index.qmd`

找到类似这段：

```markdown
I am a PhD Candidate in Economics at the University of Nottingham Ningbo China...
```

直接改成你想展示的英文介绍即可。

改完以后，在终端运行：

```bash
cd /Users/junchengzhang/Desktop/junchengzhangecon.github.io
quarto render
git add .
git commit -m "Update homepage"
git push
```

改完后同样运行：

## 例子 3：修改顶部菜单

打开：

`_quarto.yml`

找到：

```yaml
navbar:
  left:
    - text: Home
      href: index.qmd
    - text: Research
      href: research.qmd
```

这里控制顶部导航栏。一般不建议乱改缩进，因为 YAML 对空格很敏感。

## 重要提醒

- 不要直接改 `index.html`、`research.html` 这些文件。
- `.html` 文件是 Quarto 自动生成的网站成品。
- 你改了 `.html`，下一次运行 `quarto render` 会被覆盖。
- 正确修改入口永远是 `.qmd` 文件和 `_quarto.yml`。

## 本地预览

如果你想先在本地预览，不马上发布，可以运行：

```bash
cd /Users/junchengzhang/Desktop/junchengzhangecon.github.io

```

终端会显示一个本地网址，比如：

`http://localhost:xxxx`

打开它就能看到本地预览。确认没问题后，再运行 `quarto render`、`git commit`、`git push` 发布。

## 例子 4：修改 Notes / Posts 文章

现在网站有一个 `Notes` 板块，文章放在：

`posts/`

目前第一篇文章在：

`posts/first-note/index.qmd`

你要改这篇文章，就打开这个文件，修改里面的标题、日期和正文。

文章开头这一段叫 YAML 头部：

```yaml
---
title: "Why I Want to Keep Notes"
subtitle: "A small place for thoughts that are not yet papers"
date: "2026-05-16"
categories: [notes, reflection]
---
```

- `title`：文章标题
- `subtitle`：文章副标题，可以删掉或改掉
- `date`：文章日期
- `categories`：文章分类标签

正文从第二个 `---` 后面开始写。可以写中文，也可以写英文。

如果要新建第二篇文章，复制整个文件夹：

`posts/first-note/`

改成新名字，比如：

`posts/my-second-note/`

然后修改里面的 `index.qmd` 即可。
